## Especially welcome

**Methodological objections.** The [methods record](/methods/pipeline/) exists
so the reasoning can be attacked. If a resolution rule looks wrong, a threshold
looks arbitrary, or a chart supports an inference it should not, I want to know.
Corrections will be recorded and credited.

**Errors in the data.** A citation attributed to the wrong work, a passage that
does not exist, a DOI that resolves somewhere unexpected. Please include the
work and locus, and a link to the page where you saw it.

**Journals.** If you edit or publish a journal you would like represented here,
please write. To be straightforward about what I can promise: the corpus came
through a JSTOR Text Analysis Support agreement, so I cannot simply add titles.
Expanding coverage means going back to JSTOR and to the publisher. An expression
of interest from an editor carries considerably more weight in that conversation
than a request from me alone, so hearing from you is genuinely useful.

**Feature ideas.** Some things already under consideration:

- Deep links from a passage to the text on Perseus, so a locus opens the Greek
  and a public-domain translation alongside the citation counts.
- Co-citation: which passages tend to be cited *together* in the same article.
- Coverage beyond journal articles, if the licensing can be arranged.

If something else would make this useful for your work, say so. The tool exists
because passage-level questions were hard to ask, and I would rather hear which
questions are still hard.

## What I cannot do

- Provide the underlying JSTOR full text. It is confidential under the delivery
  agreement and does not leave the local machine.
- Add journals unilaterally, as above.
- Reproduce article content. This is a finding aid; it links out to JSTOR, which
  is where the scholarship should be read.
